Installation - Windows

CHECKING FOR INSTALLATION
CMake may already be installed on your system for several reasons: your operating system might come with CMake pre-installed, a software tool you installed could include CMake as a dependency, or you might have installed CMake previously and forgotten about it. Regardless of the reason, if you want to check whether CMake is installed—whether to verify an existing installation or confirm a new one—you can use the following command on Windows:

CMD - PowerShell

cmake --version

INSTALLING CMAKE
The easiest way to install CMake on a Windows machine is to download the official installer from the CMake download page. Once downloaded, double-click the installer and follow the installation wizard to customize CMake settings according to your preferences. It is generally recommended to stick with the default settings. After completing the installation, you should be able to use CMake from the command line. To confirm that CMake is properly installed, refer back to the section of CHECKING FOR INSTALLATION.

Installation - MacOS

CHECKING FOR INSTALLATION
CMake may already be installed on your system for several reasons: your operating system might come with CMake pre-installed, a software tool you installed could include CMake as a dependency, or you might have installed CMake previously and forgotten about it. Regardless of the reason, if you want to check whether CMake is installed—whether to verify an existing installation or confirm a new one—you can use the following command on MacOS:

Zsh - Bash

cmake --version

INSTALLING CMAKE
The easiest way to install CMake on a MacOS machine is by using the Homebrew package manager. First, ensure that Homebrew is installed. You can find the installation command at the top of the Homebrew homepage; here it is for your convenience:

Zsh - Bash

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

After installing Homebrew, search for a package named cmake. You should find the corresponding command to download the CMake package. I also provide the command here for your convenience:

Zsh - Bash

brew install cmake

If you prefer not to use the Homebrew package manager, you can visit the CMake download page for alternative methods to install CMake on MacOS. Once the installation is complete, you can confirm it by referring to the CHECKING FOR INSTALLATION section.

Installation - Linux

INSTALLING CMAKE
Installing CMake on a Linux-based system can be a bit challenging due to the variety of distributions, each with its own package manager. To make things easier, I’ve provided the necessary commands for the most commonly used distributions below (Please remember to update the package manager first before attempting to install CMake itself):

DEBIAN/UBUNTU
For Debian-based distributions, which use the apt (Advanced Package Tool) package manager, use the following command to update the package manager and install CMake simultaneously: 

sudo apt update && sudo apt install cmake -y

Fedora
For Fedora-based distributions, which use the dnf (Dandified YUM) package manager, use the following command to update the package manager and install CMake simultaneously: 

sudo dnf check-update --refresh && sudo dnf install cmake -y

Arch
For Arch-based distributions, which use the pacman package manager, use the following command to update the package manager and install CMake simultaneously: 

sudo pacman -Sy cmake --noconfirm
