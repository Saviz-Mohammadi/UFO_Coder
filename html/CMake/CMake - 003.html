Update - Windows

UPDATING CMAKE
CMake may already be installed on your system for several reasons: your operating system might come with CMake pre-installed, a software tool you installed could include CMake as a dependency, or you might have installed CMake previously and forgotten about it. Regardless of the reason, if you want to to update CMake on a Windows machine, then the best way is to download a newer variation of the official installer from the CMake download page.

Once downloaded, double-click the installer and follow the installer wizard to update CMake. After completing the update, you should be able to use the newer CMake version from the command line.

Update - MacOS

UPDATING CMAKE
CMake may already be installed on your system for several reasons: your operating system might come with CMake pre-installed, a software tool you installed could include CMake as a dependency, or you might have installed CMake previously and forgotten about it. Regardless of the reason, if you want to update CMake for your MacOS machine, then you first need to make sure that the Homebrew package manager is up to date by using the following command:

Zsh - Bash

brew update

Once the process is finished, you can update CMake using the following command:

Zsh - Bash

brew upgrade cmake

If you’re not using the Homebrew package manager, you can also obtain newer binaries for the macOS platform by visiting the CMake download page. You can then run the installers and update CMake.

Update - Linux

UPDATING CMAKE
Updating CMake on a Linux-based system can be a bit challenging due to the variety of distributions, each with its own package manager. To make things easier, I’ve provided the necessary commands for the most commonly used distributions below (Please remember to update the package manager first before attempting to update CMake itself):

DEBIAN/UBUNTU
For Debian-based distributions, which use the apt (Advanced Package Tool) package manager, use the following command to update the package manager and CMake simultaneously: 

sudo apt update && sudo apt upgrade cmake -y

Fedora
For Fedora-based distributions, which use the dnf (Dandified YUM) package manager, use the following command to update the package manager and CMake simultaneously: 

sudo dnf check-update --refresh && sudo dnf upgrade cmake -y

Arch
For Arch-based distributions, which use the pacman package manager, use the following command to update the package manager and CMake simultaneously: 

sudo pacman -Sy cmake --noconfirm
